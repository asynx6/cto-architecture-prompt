# VinzLeonars Core: Production-Grade AI Software Engineering Blueprint

An elite, context-aware engineering persona and quality gate blueprint designed for large-scale systems design, backend orchestration, cybersecurity hardening, and premium frontend engineering.

This repository contains the definitive configuration to transform an LLM into **VinzLeonars**, an autonomous Solutions Architect and CTO with 25+ years of experience.

## Repository Structure

* `prompt.txt`: The core system instruction block containing the complete architectural constraints, coding standards, and behavioral guidelines.
* `README.md`: This documentation file explaining the blueprint and deployment instructions.

## Deployment & Installation

### Option 1: Global System Prompt (Recommended)
1. Copy the entire contents of `prompt.txt`.
2. Navigate to your AI platform of choice (e.g., Claude Custom Instructions, OpenAI System Instructions, or Cursor/Windsurf System Prompts).
3. Paste the contents into the **System Prompt** or **Custom Instructions** text field.
4. Save the configuration.

### Option 2: Session-Based Initialization
For standalone chat sessions, copy the contents of `prompt.txt` and paste it as the very first message in a new conversation thread to initialize the persona boundaries instantly.

## Technical Breakdown (Modules 1–22)

The blueprint enforces a strict operational framework across 22 distinct architectural and behavioral modules:

### 1. Core Persona
Establishes the identity of VinzLeonars as an elite Senior Software Engineer and CTO. It forces the model to think strategically, act proactively, write production-ready code without placeholders, and automatically expand casual user input into enterprise-grade implementations.

### 2. Thinking Process (Before Writing Any Code)
Mandates a five-step cognitive chain-of-thought analysis that the AI must complete internally before generating any code: Decode Intent, Assess Impact, Architect First, Plan Minimal Change, and Execute with Precision.

### 3. Coding Standards
Enforces universal software craftsmanship rules (efficiency, Single Responsibility Principle, bundle size optimization, and the elimination of magic numbers). It includes strict backend paradigms (parameterized queries, bcrypt hashing, error boundaries) and frontend rules (mobile-first design, spring physics fluid interactions, and high-contrast professional styling).

### 4. Anti-AI-Generic-UI Law (Non-Negotiable)
Strictly bans typical "AI-generated style" patterns such as violet/purple gradients, floating neon blobs, or generic 3-column card layouts. It forces the model to match the design aesthetics of industry leaders like Stripe, Linear, and Vercel.

### 5. Strict Constraints
Prohibits the use of `any` types, `// @ts-ignore` shortcuts, code placeholders (`// TODO`), and breaking backward compatibility. It guarantees that all code generated is fully complete and heavily typed in TypeScript.

### 6. Response Format
Implements an adaptive communication wrapper. Simple fixes are returned directly, while complex features are structured into four mandatory sections: Intent Analysis, Impact Assessment, Implementation, and Side Effects.

### 7. Advanced Engineering Standards
Beds down software architecture paradigms including RESTful API envelope consistency, JWT security handling, browser storage restrictions (banning localStorage for state), and comprehensive unit testing awareness.

### 8. Elite Engineering Additions
Enforces structured logging with production context (avoiding plain strings), mandatory soft-delete schema discipline, and strict state segregation (using TanStack Query for server state instead of local React state).

### 9. Universal Language Mastery
Defines exact execution standards across multiple language stacks, including Python (strict type hinting and PEP 8), Go (explicit error handling and channel management), Kotlin/Java, Flutter, and raw SQL index optimizations.

### 10. Domain Expertise Modules
Equips the persona with advanced engineering paradigms across distinct fields: Mobile lifecycle safety, DevOps Infrastructure as Code (multi-stage non-root Docker configurations), Medallion data engineering pipelines, and reproducible AI/ML evaluation metrics.

### 11. Cybersecurity (Defensive Engineering)
Enforces a security-first engineering mindset based on the OWASP Top 10 framework, data sanitization to mitigate XSS/SQL injections, and the principle of least privilege across all system boundaries.

### 12. Code Commentary Discipline
Bans redundant comments that describe *what* the code does. Restricts commenting exclusively to explaining the business logic or architectural *why* behind complex optimizations.

### 13. Karpathy Behavioral Guidelines
Prioritizes extreme simplicity over speculative abstraction. It forces the model to minimize code length, state implicit assumptions upfront, push back against over-complication, and make precise, surgical changes to legacy codebases.

### 14. Human-Like Intelligence Layer
Enables full project contextual awareness and business logic comprehension, allowing the persona to communicate explanations clearly in Indonesian while keeping code perfectly clean in English.

### 15. Adaptive Intelligence Modes
Introduces an automatic intent-detection router that switches the AI behavior dynamically based on keywords (e.g., BUILD MODE for architecture, DEBUG MODE for root-cause analysis, REVIEW MODE for security scoring).

### 16. Self-Correction Protocol
An internal pre-flight quality gate. The model must evaluate its own output against TypeScript, security, and mobile layout checklists before rendering the final response.

### 17. Context Window Intelligence
Optimizes interaction with massive codebases by prioritizing entry points, building mental dependency graphs, and emitting clean git-diff style outputs instead of rewriting entire unmutated files.

### 18. Multi-Paradigm Thinking
Ensures the proper architectural model (OOP, Functional, Reactive, or Declarative) is chosen based on the natural requirements of the technical problem.

### 19. Production Readiness Checklist
Validates that every single technical feature delivered is ready for production scaling, covering structured logging, rollback strategies, and graceful degradation paths under load.

### 20. Single File Configuration Principle & Contextual Scope
Enforces absolute centralization of environment parameters into `.env` and `.env.example` configurations for new setups, while strictly preventing breaking changes or configuration modifications on legacy projects.

### 21. Conflict Resolution Protocol
Establishes a global priority matrix to handle competing rules, ensuring user explicit instructions and engineering simplicity always override speculative scalability or convenience.

### 22. Proactive Architecture Proposal
Forces the persona to surface exactly ONE high-impact architectural decision (e.g., WebSockets vs Polling) to the user *before* implementation begins, preventing catastrophic structural refactoring down the line.
